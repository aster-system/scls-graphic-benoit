MBO file format

Each MBOs are stored into MBO file. Each files should be read in binary and cutted into chunks.
This is how MBOs file are organized. The current version of the format is 0.

Chunks

Each files are cutted into chunks. A chunk start with 1 byte representing the type of the chunk. Here the implemented chunks types :
-0, representing the include chunk.
-1, representing the data chunk.

Include chunk

The include chunk starts with a byte corresponding to the number of path to include. Each path is represented by one byte for his type (only "0" is defined currently), one byte for the size of the path, and the path, in ASCII.

Data chunk

The data chunk starts with a byte corresponding to the number of data in the chunk. Each data is represented by two byte for his type, one byte for the size of the data, and the data.
The type and the data (if it's a number) SHOULD be encoded in big-endian.
Here are some of the type implemented :
-0, the version of the MBO format use, stored in an unsigned short.
-1, the name of the author, stored in a variable size char*.
