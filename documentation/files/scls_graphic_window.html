SCLS Graphic Window

Presentation

SCLS Graphic Window contains the main class which allows to use a window with SCLS. It is made to be very easily usable, by anybody.

Content

SCLS Graphic Window is axed around the class "Window". It's the only notable thing in this file. However, this class is full of usefull things to do a window in only some lines.

The hierarchy of the class is quite complicated. Indeed, some informations need to be accessible to all the object in SCLS Window. However, the Window class must include all the others classes. So, the Window class can't be a way to access to those informations. Actually, there are 2 mores classes : _Window_Advanced_Struct and _Window_Base_Struct. Window herits from _Window_Advanced_Struct and _Window_Advanced_Struct herits from _Window_Base_Struct. There are contained in the files "scls_graphic_window_advanced_struct.h" and "scls_graphic_window_base_struct.h".
_Window_Base_Struct contains usefull datas that should be accessible for everywhere in the program.
_Window_Advanced_Struct contains data for the object which are not include by _Window_Base_Struct.

Window page handling

To display Objects, the class uses the system of page. A page is a way to display a lot of objects easily. The class "_Page" allows to easily handle them. However, this class should be used for heritage, and not directly. As the Objects, the pages have their name, used in the Window to identify them. If you do not want to display any pages, set the current page to "", an empty std::string.

Window operating

To be usable, the window must be able to evolve in the time. For that, like in each graphic library, there are an infinite while loop, allowing to update and render the window. This loop should be created by the user of the window. One loop iteration is called a frame.
In Window, a FPS is the number of frame in one second, and is calculated by "update_event()".

_Page methods

virtual void render()
Render the _Page.
It is recommended to use this function in heritage.
virtual void update()
Update each objects in the _Page.
It is recommended to use this function in heritage.

Window methods

virtual void render()
Render the window graphic.
It is recommended to call this function after all the others function of the frame.
virtual void update()
Update each objects in the Window.
virtual void update_event()
Update the system and Window events.
It is recommended to call this function before all the others function of the frame.

Window input handling

Like every software, Window has a system allowing to handle the input coming from the system or the user.
As GLFW works with a lot of callback functions, a lot of functions are hidden to be usable with a class system.
The most important input features is of course present : the keyboard. The window registers each keys as the character that they represents.
Window allows to have some datas about the cursor and the mouse. You can have the position of the mouse, if it is clicked, its texture, or even if it is displayed or not.
The resizing of the window is also an input.

Window model storage

One of the key features made possible by the use of _Window_Advanced_Struct is the possibility to handle models directly in the Window. Indeed, the Window can store the necessary things to create objects. It contains Textures, VAOs, VBOs and Shader_Programs. It provides a good access to it, without being very complicated. Each of this objects has an unique name, and are usable from it.
All this functions are implemented into _Window_Advanced_Struct.
