SCLS Graphic Window

Presentation

SCLS Graphic Window contains the main class which allows to use a window with SCLS. It is made to be very easily usable, by anybody.

Content

SCLS Graphic Window is axed around the class "Window". It's the only notable thing in this file. However, this class is full of usefull things to do a window in only some lines.

The hierarchy of the class is quite complicated. Indeed, some informations need to be accessible to all the object in SCLS Window. However, the Window class must include all the others classes. So, the Window class can't be a way to access to those informations. Actually, there are 2 mores classes : _Window_Advanced_Struct and _Window_Base_Struct. Window herits from _Window_Advanced_Struct and _Window_Advanced_Struct herits from _Window_Base_Struct. There are contained in the files "scls_graphic_window_advanced_struct.h" and "scls_graphic_window_base_struct.h".
_Window_Advanced_Struct contains data for the object which are not include by _Window_Base_Struct.

Window main functions

Window has some must-see features, made to be simple but very usefull.
You can gives the window a special background color, even if a page could hide it.
You can also handle if the window is running or not easily.

Functions

Window(int window_width, int window_height, std::string exec_path)
The most basic window constructore, which should be easily creatable for the user.
Color background_color()
Returns the background color of the window.
bool is_running()
Returns if the window should continue to run or not.
If you want to do a infinite loop for your program, DO NOT use this function as the condition. Use run() instead.
void load_from_config_file(std::string path)
Load the window from a portable configuration file at the path "path".
bool run()
Returns if the window is still running or not.
When it returns false, GFLW is terminated.
If you want to do a infinite loop for your program, use this function as the condition.
void set_background_color(Color new_background_color)
Set the background color to "new_background_color".
void set_is_running(bool new_is_running)
Set if the window should continue to run or not by "new_is_running".
If you set it to false, the window will close in the next frame.
GLFWwindow* window()
Returns a pointer to the GLFW window.

Window page handling

To display Objects, the class uses the system of page. A page is the first object rendered, which should have a list of childrens to display. As the Objects, the pages have their name, used in the Window to identify them. If you do not want to display any pages, set the current page to "", an empty std::string.

Methods

bool contains_page(std::string name)
Returns if the window contains the page called "name" or not.
Object* current_page()
Returns a pointer to the current page.
If the current page is "" or is not existing, reutrns 0.
std::string current_page_name()
Returns the name of the currently displayed page.
Object* page(std::string name)
Returns the pointer to the page Object called "name".
If the window does not contains "name", it prints a warning and returns 0.
std::map<std::string, Object*>& pages()
Returns a reference to the std::map of the pages in the window.
void set_current_page(std::string new_current_page)
Set the current page to "new_current_page".
If the window does not contains "new_current_page", it prints a warning and does not change the current page.

Window operating

To be usable, the window must be able to evolve in the time. For that, like in each graphic library, there are an infinite while loop, allowing to update and render the window. This loop should be created by the user of the window. One loop iteration is called a frame.
In Window, a FPS is the number of frame in one second, and is calculated by "update_event()".

Methods

virtual void render()
Render the window graphic.
It is recommended to call this function after all the others function of the frame.
virtual void update()
Update each objects in the Window.
virtual void update_event()
Update the system and Window events.
It is recommended to call this function before all the others function of the frame.

Window input handling

Like every software, Window has a system allowing to handle the input coming from the system or the user.
As GLFW works with a lot of callback functions, a lot of functions are hidden to be usable with a class system.
The most important input features is of course present : the keyboard. The window registers each keys as the character that they represents.
Window allows to have some datas about the cursor and the mouse. You can have the position of the mouse, if it is clicked, its texture, or even if it is displayed or not.
The resizing of the window is also an input.

Methods

void hide_cursor()
Hide the cursor to the user.
void resize_window(unsigned int width, unsigned int height)
Resize the window at the size ("width", "height").
void show_cursor()
Show the cursor to the user.

Hidden functions

void _framebuffer_size_callback(GLFWwindow* window, int width, int height)
Callback function when the window is resized.
"width" and "height" are assignated to two hiddens variables : "int _global_screen_width" and "int _global_screen_height", which are the destination of references into Window.
void _load_keys()
Load every keys supported in the window.

Window model storage

One of the key features made possible by the use of _Window_Advanced_Struct is the possibility to handle models directly in the Window. Indeed, the Window can store the necessary things to create objects. It contains Textures, VAOs, VBOs and Shader_Programs. It provides a good access to it, without being very complicated. Each of this objects has an unique name, and are usable from it.
All this functions are implemented into _Window_Advanced_Struct.

Methods

void add_vbo(std::string name, VBO* vbo)
Add an existing VBO "vbo" in the Window with the name "name".
bool contains_texture(std::string name)
Returns if the Window contains a Texture called "name";
bool contains_vao(std::string name)
Returns if the Window contains a VAO called "name";
bool contains_vbo(std::string name)
Returns if the Window contains a VBO called "name";
Texture* texture(std::string texture_name, bool copy_texture = false)
Returns the texture named "name" in the Window. If the window does not has a "texture_name" texture, it returns 0.
If "copy_texture" is true, the method copy the texture and returns the copy to not alterate the original texture.

_Window_Advanced_Struct hiddens things

To work correctly, _Window_Advanced_Struct has some hiddens things. You (theorically) do not have to use it, but they are either here.

Methods

_Window_Advanced_Struct(double& mouse_x, double& mouse_y, int& window_width, int& window_height, std::string exec_path)
Only _Window_Advanced_Struct constructor, taking references to the mouse position ("mouse_x", "mouse_y") and the window size ("window_width", "window_height"). It also takes the path "exec_path" of the program.
virtual ~_Window_Advanced_Struct()
_Window_Advanced_Struct destructor, freeing the memory from the things created with "new".
