SCLS Graphic Object

Presentation

SCLS Graphic Object contains the base objects usable in SCLS Graphic.

Content

SCLS Graphic Object is axed around the class "Object". It's the only notable thing in this file. However, this class is full of usefull things to handle objects in the window.

Object main functions

Object has some must-see features, made to be simple, but very usefull.
Each Object has an unique name, made to be easily differentiable of the others Object.
You can also easily hide or show any object by making it visible or not. It's a very important feature to make a software properly.
An another thing that can describe the Object is its tags. They are not unique and can be any std::string.
To evolve in the space, the object uses a Transform_Object.
As everything in SCLS Graphic, the Object has a pointer to the main window struct.

Functions

Object(Advanced_Struct* window_struct)
Most basic constructor of an Object, used to clone some objects if necessary. It is contained into the Window "window_struct".
Object(Advanced_Struct* window_struct, std::string a_name, std::string texture_name, std::string vao_name)
Simple Object constructor, usable by the developer. The name of the object is "a_name". It is contained into the Window "window_struct".
The Texture applied is the main texture "texture_name", and the VAO applied is the main VAO "vao_name". See the graphic part for more informations.
bool contains_tag(std::string tag)
Return if the object contains the tag "tag" in its tags.
_Window_Advanced_Struct* window_struct()
Return the pointer to the main window struct .
std::string name() const
Return the name of the object.
void set_visible(bool new_visible)
Change the visiblity of an Object by "new_visible".
std::vector<std::string>& tags()
Return the std::vector of each tags of the Object.
Transform_Object* transform()
Return a pointer to the Transform_Object of this object.
bool visible() const
Return if an Object is visible or not.
~Object()
Most basic destructor of an Object, deleting the Transform_Object of this object.

Object graphics functions

It's may be trivial, but the graphic Object needs graphic functions. This functions are made to not be used by the user if it's not necessary.
It allows the handle of the VAO and the Texture of this object.

Functions

virtual void render()
Render the object on the window.
This function may be heritated if necessary.
Texture* texture()
Return the pointer to the Texture used by this object.
VAO* vao()
Return the pointer to the VAO used by this object.

Object heritage functions

To be the most modulable possible, this class needs a good heritage handling. To respect this, it has a lot of heritage functions, callable by the children classes.

Functions

virtual void after_loading()
Function called after the loading of the window.
virtual void last_update()
Last update-like function called during a frame.
virtual void update()
First update-like function called during a frame.
