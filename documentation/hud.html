Matix - HUD

Presentation

An HUD represents the 2D interface directly drawn on the screen, without 3D interference.

Displaying

The displaying of each objects is clearly defined.

Texturing

Each object can have (or not have) a texture to display on the screen.
If the object has not any texture, it should be create from Scratch by the object.
The texture can be moldable or not, and can have it's own transformation.

Border

An object can have a border, by shrinking the texture if moldable.
The border can have any RGBA color.

Transformation system

The HUD transformation system uses positioning, rotating then scaling.

Hierarchy

Each HUD objects has a parent. The parent position corresponds to the origin of the child transformation plane.
The scale of the parent represents the (1, 1) of the position of the child.
The absolute position of the child is the addition of each absolutes positions of every parents of it, considering the scaling.
So, it represents the REAL position of the object in the global window.
The absolute scale of the child is the addition of each absolutes scales of every parents of it.
So, it represents the REAL scale of the object in the global window.

Positioning

The positioning system take as units a purcentage of the screen size as units.
The exact position of the object is the CENTER of the object.
However, a function can return a more-UI like pixel rect.
It is possible to convert pixel to purcentage of the screen and use pixels to locate an object, then converted to purcentage.
However, it is MANDATORY that the state of the window when the function is used represents the correct position of the object.
So, it's recommended to use this functions after every start windows modification.

Rotating

The rotating system take as units an angle in degrees.
The objects can rotate around the 3 axes, even if it's not very usefull.

Scaling

The scaling system take as units a purcentage of the screen size as units.
For a 16/9 parent, the scale (16/9, 1) represents the entire parent covered.
The width correspond to the parent ratio and the height is always one.
If the texture is moldable, it fits perfectly in the scaled object.
If it's not, it stays at the center of the object unchanged.
Anything can change the purcentage of the texture other that the exterior of the object.
